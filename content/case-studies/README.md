# Case Study Markdown Format

All case studies should follow this frontmatter structure:

```yaml
---
title: "Project Title"
client: "Client Name or Type"
category: "Service Category"
date: "YYYY-MM"
duration: "X weeks/months"
technologies: ["Tech1", "Tech2", "Tech3"]
outcome: "Main measurable outcome"
featured: true/false
---
```

## Content Structure

1. **Challenge** - Client's problem, pain points, business context
2. **Solution** - Your approach, key features, technical implementation
3. **Technical Highlights** - Code samples, architecture decisions
4. **Results** - Metrics, ROI, business impact with specific numbers
5. **Client Testimonial** - Quote from client (if available)
6. **Key Learnings** - Insights gained from the project
7. **Tech Stack** - Full list of technologies used

## Files Included

- `customer-support-ai-agent.md` - AI Agent case study
- `saas-landing-page.md` - Landing page conversion optimization
- `ai-product-launch-campaign.md` - AI-powered marketing campaign

## Using These Files

1. **Update frontmatter** with real project details
2. **Replace placeholder content** with actual project information
3. **Add real metrics** from your projects
4. **Include client testimonials** if you have them
5. **Update tech stack** to match what you actually used
6. **Add screenshots** or images if available

## Adding New Case Studies

Create new `.md` files in this directory following the same structure.
